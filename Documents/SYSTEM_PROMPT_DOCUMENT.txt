    3. Program Flow
        The program flow (w skrócie):
            1. Ask for a department name to create a shared mailbox for, e.g. 'Finances'.
            2. Verify the name using regex. 
            3. If it is correct, check whether account with added prefix (e.g. 'sm_finances') already exists in the AD.
            4. If not, create an account (AD User) with that name.
            5. After account, create an access group (AD Group) with the name 'sg_sm_finances_access'.
            5. After access group is created, check whether there is a group in AD with the department name
            from the 1 point - 'Finances'.
            6. If there is, ask user if he wants to add this group to the newly created access group 'sg_sm_finances_access'.
            7. Whether user add the found deparment group or not, in the next step (przekieruj go) to the main menu.
            8. In the main menu user have a few options to choose for deployment process completion.
                The options are:
                    * "Add Members or Groups",
                    * "Remove Members or Groups",
                    * "Show Group Members",
                    * "Exit"
            9. After user choose "Exit" option, terminate the program.
